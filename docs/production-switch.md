# Перевод на продуктивные каналы

## Что нужно изменить

Все настройки подключений хранятся в одном файле `.env` на сервере. Для перехода на рабочие каналы нужно заменить тестовые ID на реальные — никаких изменений в коде не требуется.

---

## Шаг 1 — Узнать ID рабочих каналов

### Telegram канал

1. Добавьте бота `@domik_posting_bot` в администраторы рабочего канала (права: публикация сообщений)
2. ID канала — число вида `-100xxxxxxxxxx`
3. Узнать ID: перешлите любое сообщение из канала в бот `@userinfobot` — он ответит с ID

### Telegram группа

1. Добавьте бота в администраторы рабочей группы (права: публикация сообщений)
2. ID группы узнаётся аналогично через `@userinfobot`

### ВКонтакте

1. ID сообщества — число в адресной строке: `vk.com/public`**`123456`**
2. Токен сообщества: Управление → Работа с API → Ключи доступа → Создать ключ
   - Обязательные права: **стена**, **фотографии**

---

## Шаг 2 — Обновить настройки на сервере

Подключитесь к серверу и откройте файл настроек:

```bash
ssh root@201.51.6.88
nano /opt/domik-posting/.env
```

Найдите и замените строки:

```env
# Telegram — вставьте ID рабочих каналов
TELEGRAM_BOT_TOKEN=<токен бота — не меняется>
TELEGRAM_CHANNEL_ID=-100XXXXXXXXXX    # ← заменить
TELEGRAM_GROUP_ID=-100XXXXXXXXXX      # ← заменить

# ВКонтакте — вставьте данные рабочего сообщества
VK_COMMUNITY_TOKEN=vk1.a.XXXXXXXXXX  # ← заменить
VK_GROUP_ID=XXXXXXXXXX               # ← заменить
```

Сохраните файл: `Ctrl+O` → `Enter` → `Ctrl+X`

---

## Шаг 3 — Перезапустить приложение

```bash
cd /opt/domik-posting
docker compose up -d --force-recreate app
```

Новые настройки вступают в силу немедленно.

---

## Шаг 4 — Проверить

1. Войдите на https://post.domik-l.spb.ru
2. На дашборде в блоке **Платформы** убедитесь, что нужные платформы отображаются с **зелёной точкой**
3. Создайте тестовый пост и убедитесь, что он опубликован в рабочих каналах

## Если что-то пошло не так

Посмотреть логи приложения:

```bash
docker logs domik-posting-app-1 --tail 50
```

Вернуться к предыдущим настройкам: открыть `.env`, восстановить старые значения, снова выполнить `docker compose up -d --force-recreate app`.
