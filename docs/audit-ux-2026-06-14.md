## КРИТИЧЕСКИЕ ПРОБЛЕМЫ (мешают работе)

### 1. Нет лимитов символов с визуальным предупреждением
**Файл:** `approve.html`

Счётчик символов есть, но нет индикации лимитов платформ. Редактор может отправить пост на 6000 символов в Одноклассники (лимит 3000) и узнает об ошибке только после публикации.

Лимиты платформ:
- Telegram: 4096 символов
- VK: 16384 символов
- Одноклассники: 3000 символов
- Яндекс: 4096 символов

**Решение:** Добавить константы лимитов и смену цвета счётчика при превышении:
```js
const LIMITS = { telegram_channel: 4096, telegram_group: 4096, vk: 16384, ok: 3000 };
function updateCount(id, platform, text) {
  const el = document.getElementById('cc-' + id);
  const limit = LIMITS[platform];
  el.textContent = text.length + (limit ? ' / ' + limit : '') + ' симв.';
  el.className = (limit && text.length > limit)
    ? 'text-danger fw-bold' : 'text-muted';
}
```

### 2. Мобильный sidebar не закрывается кликом вне него
**Файл:** `base.html`

Нет overlay/backdrop и нет закрытия при клике по контенту. Пользователь «застревает» с открытым меню.

**Решение:**
```js
document.addEventListener('click', function(e) {
  const sidebar = document.getElementById('sidebar');
  if (sidebar.classList.contains('open') &&
      !sidebar.contains(e.target) &&
      !e.target.closest('[onclick*="sidebar"]')) {
    sidebar.classList.remove('open');
  }
});
```

### 3. Форма загрузки не валидирует файл/текст на клиенте
**Файл:** `content_new.html`

При отправке без файла форма уходит на сервер, spinner запускается, кнопка блокируется — пользователь ждёт 10–20 секунд холостого запроса к GigaChat.

**Решение:** Клиентская валидация перед отправкой:
```js
document.getElementById("uploadForm").addEventListener("submit", function(e) {
  const type = document.querySelector('[name=content_type]:checked').value;
  if (['photo','photo_text','video','audio'].includes(type)) {
    if (!document.getElementById('fileInput').files.length) {
      e.preventDefault();
      // показать inline-ошибку
      return;
    }
  }
  if (type === 'text' && !document.querySelector('[name=text]').value.trim()) {
    e.preventDefault(); return;
  }
});
```

### 4. Нет блокировки кнопки после сабмита — возможны дублирующие публикации
**Файл:** `approve.html`

Публикация в соцсети занимает 5–10 секунд. Нетерпеливый пользователь нажмёт дважды — получит дублирующий пост.

**Решение:**
```js
document.querySelector('form').addEventListener('submit', function() {
  const btn = document.querySelector('[value=approve]');
  btn.disabled = true;
  btn.innerHTML = '<span class="spinner-border spinner-border-sm me-1"></span> Публикуем...';
});
```

### 5. Двойной атрибут `style=""` на одном элементе (баг)
**Файл:** `content_new.html`

```html
<div id="loadingMsg" style="display:none;" class="mt-3 text-muted" style="font-size:.9rem;">
```
Второй `style` игнорируется браузером. `font-size:.9rem` никогда не применится.

**Решение:** Объединить в один атрибут: `style="display:none; font-size:.9rem;"`

### 6. Bootstrap Icons подключён в конце `<body>` в login.html

Иконка в сообщении об ошибке может не отображаться при медленной загрузке.

**Решение:** Перенести `<link>` Bootstrap Icons в `<head>`.

## ВАЖНЫЕ УЛУЧШЕНИЯ

### 7. Нет хлебных крошек
**Файлы:** `content_preview.html`, `approve.html`

Flow: Новый материал → Черновики → Согласование. Пользователь не видит на каком шаге находится.

**Решение:** Добавить breadcrumbs или stepper в заголовок страниц:
```html
<nav aria-label="breadcrumb">
  <ol class="breadcrumb" style="font-size:.83rem;">
    <li class="breadcrumb-item"><a href="/dashboard">Дашборд</a></li>
    <li class="breadcrumb-item active">Черновики #{{ item.id }}</li>
  </ol>
</nav>
```

---

### 8. Страница предпросмотра дублирует согласование (лишний шаг)
**Файл:** `content_preview.html`

Черновики отображаются в read-only режиме. Пользователь вынужден делать лишний переход на approve для редактирования.

**Решение:** Объединить preview и approve в одну страницу, или сделать тексты редактируемыми прямо на preview.

### 9. Кнопка «Согласовать» видна редакторам
**Файл:** `dashboard.html`

Кнопка показывается всем пользователям без проверки роли.

**Решение:**
```html
{% if user.role == 'approver' %}
  <a href="/posts/approve/{{ item.id }}" ...>Согласовать</a>
{% else %}
  <a href="/content/{{ item.id }}/preview" ...>Просмотр</a>
{% endif %}
```

### 10. Timezone bug в форме планирования
**Файл:** `approve.html`

`d.setHours(d.getHours() + 4)` — хрупкое решение. `datetime-local` без явного timezone возвращает локальное время браузера, которое сервер интерпретирует как UTC. Публикация уйдёт на 3 часа раньше.

**Решение:** Явно указывать UTC в подсказке, хранить всё в UTC, показывать текущее московское время динамически.

### 11. Таблица дашборда не адаптирована под мобильный
**Файл:** `dashboard.html`

5 колонок на экране 360px — горизонтальный скролл.

**Решение:** Скрыть колонку «Кто загрузил» на мобильном (`d-none d-md-table-cell`), ограничить preview текста 40 символами.

### 13. Нет предупреждения при уходе с несохранёнными правками
**Файл:** `approve.html`

Пользователь отредактировал тексты и случайно нажал «На главную» — все правки потеряны.

**Решение:**
```js
let isDirty = false;
document.querySelectorAll('.post-textarea').forEach(ta => {
  ta.addEventListener('input', () => isDirty = true);
});
window.addEventListener('beforeunload', (e) => {
  if (isDirty) { e.preventDefault(); e.returnValue = ''; }
});
document.querySelector('form').addEventListener('submit', () => isDirty = false);
```
